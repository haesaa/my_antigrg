# 01-input-gate: 입력 게이트 스킬

> **실행자**: Leader (직접 실행)
> **실행 시점**: 유저가 프로젝트를 시작할 때
> **쓰기 권한**: `input/`, `PIPELINE-LOG.md`

## 목적

유저가 제공한 입력 파일의 유형을 분석하여 MV-Story 모드 또는 Direct 모드를 자동 확정하고, 해당 모드에 맞는 필수 에셋을 검증한 뒤 A1 에이전트로 핸드오프합니다.

## 필수 입력 파일

**[MV-Story 모드] 필수 입력**

| 파일 | 설명 | 검증 기준 |
|------|------|----------|
| `music-meta.md` 또는 `*.mp3` | 음악 메타데이터 또는 원본 음원 | BPM, Key, 구간 표 포함 |
| 표정시트 이미지 (`*표정*` 또는 `*3x4*`) | 캐릭터 표정 그리드 | 이미지 파일, 1MB 이상 |
| 캐릭터시트 이미지 (`*캐릭터*` 또는 `*character*`) | 캐릭터 외형 레퍼런스 | 이미지 파일, 1MB 이상 |

**[Direct 모드] 필수 입력**

| 파일 | 설명 | 검증 기준 |
|------|------|----------|
| 프레임 지시서 (텍스트) | 씬별 화면/감정/카메라/동선 기술 | `## scene-XX` 헤더 1개 이상 |
| 캐릭터시트 이미지 (`*캐릭터*` 또는 `*character*`) | 캐릭터 외형 레퍼런스 | 이미지 파일, 1MB 이상 |

**[공통 선택 입력]**

| 파일 | 설명 |
|------|------|
| `synopsis.md` / `synopsis.txt` | 시놉시스 또는 세계관 설정 |
| `style-guide.md` | 영상 연출 스타일 가이드 |

## 워크플로우

### Step 1: 디렉토리 스캔 및 파일 탐지

- `input/` 디렉토리를 스캔하여 위 필수/선택 파일의 존재 여부를 확인합니다.
- 파일명 패턴 매칭(한글/영문 모두 지원)으로 각 카테고리를 자동 분류합니다.

### Step 2: 유효성 검증

- **음악 메타**: `music-meta.md`의 BPM, Key, 구간 구조 표가 파싱 가능한지 확인합니다.
- **표정시트**: 이미지 해상도가 그리드 분할에 충분한지(최소 600×400) 확인합니다.
- **캐릭터시트**: 이미지 파일 크기가 0바이트가 아닌지 확인합니다.

```
모드 판정 기준:
- music-meta.md 또는 mp3 파일 + 표정시트 존재 → MV-Story 모드 확정
- 프레임 지시서 텍스트 + 캐릭터시트만 존재 → Direct 모드 확정
- 양쪽 판단 불가 → 유저에게 모드 선택 질의 후 진행

모드 확정 후:
- PIPELINE-LOG.md에 확정된 모드와 감지된 파일 목록 기록
- MV-Story → A1이 SKILL-1A부터 실행
- Direct → A1이 SKILL-1B부터 실행 (1A 스킵)
```

### Step 4 (선택, MV-Story 모드 한정): T1-SKIP 판정

- `music-meta.md`가 이미 존재하는 경우 오디오 분석 단계를 스킵(`[T1-SKIP]`)하고 메타 데이터를 바로 JSON 변환합니다.

## 예외사항

1. 필수 파일 누락 시: 누락된 파일 목록을 유저에게 명시적으로 알리고 파이프라인을 중단합니다.
2. 파일명 인코딩 이슈: 한글 파일명을 영문 별칭으로 복사하여 후속 단계의 호환성을 확보합니다.
3. 모드 판정 불가 시: music-meta와 프레임 지시서가 동시에 존재하면 유저에게 "어느 모드로 진행할까요?" 질의 후 진행.
